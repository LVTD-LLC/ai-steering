# VISION.md

## Vision

AI Steering Files should become the definitive practical resource for choosing and writing repository steering files for AI coding agents.

It should be useful to a human browsing the web, a maintainer reviewing a pull request, and an AI agent reading the repo before making changes.

## Durable Principles

- Practical beats comprehensive. Prefer guidance someone can apply in a real repo today.
- Repo context beats generic AI advice. Every entry should explain how the file changes day-to-day agent work.
- Primary sources matter. Link to official docs and strong examples when possible.
- Short files are better than exhaustive files. Steering files should reduce uncertainty, not create another documentation maze.
- Vendor-neutral guidance should stay separate from tool-specific syntax.
- The site and README should come from the same source of truth.

## What This Project Should Become

- A compact catalog of the most useful steering files.
- A living example repo that uses the files it recommends.
- A reliable place to compare agent-agnostic and agent-specific instruction files.
- A source maintainers can link when explaining what belongs in repo context.

## Non-Goals

- Do not become a general AI tools directory.
- Do not rank model quality or coding assistant vendors.
- Do not mirror long vendor documentation.
- Do not add dynamic product features before the static reference is excellent.
- Do not optimize for flashy presentation over scannable technical guidance.

## Success Signals

- Readers can decide what to add to their own repo without needing a sales-style explanation.
- Agents working in this repo can follow the local instructions without asking basic setup questions.
- New catalog entries are easy to review because the structure is consistent.
- Examples and links are curated enough that users trust them.
